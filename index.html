Change Request Rotation

Option 1 (full rotation)

Monday Tuesday Wednesday Thursday Friday
Group A New Organizations Data Correction User Updates Software Issue Permission Updates
Group B Permission Updates New Organizations Data Correction User Updates Software Issue
Group C Software Issue Permission Updates New Organizations Data Correction User Updates
Group D User Updates Software Issue Permission Updates New Organizations Data Correction
Group E Data Correction User Updates Software Issue Permission Updates New Organizations
  • We break up into 5 groups
  • each group tackles a different pile each day

Option one is pretty straight forward. This way we are always hitting each request type, but we also aren't always doing the same ones every day. This one is my overall favorite.

Option 2 (half and half)

Monday Tuesday Wednesday Thursday Friday
Group A New Organizations Data Correction User Updates Software Issue Permission Updates
Group B Normal Tickets
  • We Split into two groups
  • One half takes on specific ticket types
  • The other half works as normal

This is pretty similar to Option 1, but a little with more people on a given subject.

Option 3 (On and Off)

Monday Tuesday Wednesday Thursday Friday
Group A New Organizations Normal Ticket User Updates Normal Tickets Permission Updates
Group B Permission Updates Data Correction Software Issue
Group C Software Issue New Organizations User Updates
Group D User Updates Permission Updates Data Correction
Group E Data Correction Software Issue New Organizations
  • We split into 5 teams again
  • Take two (or three) days where we just work specific tickets
  • The other days we work normal tickets

This feels like the least ideal plan but I wanted to include it for variety

Option 4 (Lunch Break Switch)

Monday Tuesday Wednesday Thursday Friday
Group A New Organizations Data Correction User Updates Software Issue Permission Updates
Normal Tickets
Group B Permission Updates New Organizations Data Correction User Updates Software Issue
Normal Tickets
Group C Software Issue Permission Updates New Organizations Data Correction User Updates
Normal Tickets
Group D User Updates Software Issue Permission Updates New Organizations Data Correction
Normal Tickets
Group E Data Correction User Updates Software Issue Permission Updates New Organizations
Normal Tickets
  • We break up the day into two sections
  • each group tackles a different pile for 1/2 of each day
  • the other 1/2 of the day is normal mode

This is basiclly just option 1 but we split the day in half. The idea here is to achieve 4 hours each day of concentrated work where there is no meeting and you block out Webex and Outlook.
